* 관리자 메뉴 재고추가
1. 관리자 로그인(메뉴에 문자 입력)
2. 재고 2개 추가(추가할 때 갯수 음수나 문자 입력)> 입력 오류 나는지 확인
3. 재고 출력                                             >정상적으로 입력됬는지 확인
4. 로그아웃

*회원가입
1. 회원가입(guest1)
2. 같은 아이디로 회원가입 시도                      >중복아이디로 회원가입이 되는지
3. 회원가입(guest2)

*회원1 로그인
1. guest1으로 로그인
2. 장바구니에 책1 담기 - 재고보다 많이 넣거나 음수입력, 문자입력  >입력오류체크
3. 장바구니에 책1 다시 담아보기                                      >장바구니에 같은것이 들어있으면 일어나는 반응체크
4. 장바구니에 책2 담기 - 없는 코드나 한글입력해보기                 >입력오류체크
5. 장바구니에 책3 담기
6. 장바구니에 책2 담은거 삭제하기                                       
7. 책4 직접구매                                                     >재고량보다 적게(목록이 삭제안되고 갯수만 줄어드는지)
>회원1 은 책1,3 장바구니, 책4 구매신청

(장바구니 책1, 책2, 책3 담기
>책2 삭제, 책4 직접구매 )

*회원2 로그인
1. 로그아웃 후 guest2 로 로그인
2. 장바구니에 책1 담은 후 구매                                  > 재고량 전부(책4가 갯수가 줄어있는지 목록으로확인)
3. 구매중인 목록 확인
4. 로그아웃
>회원2 책1 구매신청

*host 
(알림확인)
1. host 로그인
2. 주문관리 id별 > 회원2의 책1 승인처리
2. 주문관리 갯수별 > 책4 취소처리(회원1의)
3. 결산                                                            >승인처리 된것만 계산되는지           
4. 재고 수정 > 책3 수정(갯수나 가격에 음수나 문자 입력) & 삭제 >손님 장바구니에 있던 책이었다면 일어나는 반응체크
5. 로그아웃

*회원1 로그인
(알림확인)
1. 장바구니 - 목록확인
2. 장바구니 책 1 > 구매 해보기 (잔고 없으면 어떻게 처리되는지)>회원2가 재고를 전부 샀기 때문에 어떻게 반응하는지
3. 로그아웃 

*회원 2 로그인
1. 책 1 구매했던거 환불해보기                            >구매했던 목록 제대로 출력되는지

(알림확인)
1. 재고 늘어났는지 확인
